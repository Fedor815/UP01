# Flowchart: Процесс парсинга медицинских данных

```mermaid
graph TD
    A[Начало парсинга] --> B{Есть ли сайты<br/>для парсинга?}
    B -->|Да| C[Загрузка списка сайтов]
    B -->|Нет| Z[Завершение]
    
    C --> D[Для каждого сайта]
    D --> E{Сайт доступен?}
    E -->|Да| F[Отправка HTTP запроса]
    E -->|Нет| G[Логирование ошибки<br/>пропуск сайта]
    
    F --> H[Парсинг HTML]
    H --> I[Извлечение данных<br/>врачи, цены, расписание]
    I --> J[Валидация данных]
    J --> K{Данные валидны?}
    K -->|Да| L[Сохранение в БД]
    K -->|Нет| M[Логирование ошибки<br/>данные не сохранены]
    
    L --> N[Обновление статистики]
    G --> N
    M --> N
    
    N --> O{Остались сайты?}
    O -->|Да| D
    O -->|Нет| P[Отправка отчёта<br/>в Telegram]
    P --> Q[Обновление статуса<br/>в системе]
    Q --> Z
